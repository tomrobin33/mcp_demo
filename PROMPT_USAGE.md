# 系统提示词使用说明

## 概述

本项目提供了两个系统提示词文件，用于指导AI助手进行智能文档生成：

1. **`system_prompt.md`** - 完整的系统提示词（详细版）
2. **`ai_assistant_prompt.md`** - 简化的AI助手提示词（精简版）

## 文件说明

### system_prompt.md（详细版）
- **用途**：完整的系统文档和参考指南
- **内容**：包含完整的工作流程、模板库、质量保证机制等
- **适用场景**：系统开发、培训、深度定制
- **新增功能**：内容过滤指南，避免敏感内容

### ai_assistant_prompt.md（精简版）
- **用途**：AI助手的实际工作指导
- **内容**：核心工作流程、工具调用模板、响应格式等
- **适用场景**：日常AI助手使用、快速部署
- **新增功能**：内容过滤提醒，确保内容安全

## 使用方法

### 1. 在AI助手中使用

将 `ai_assistant_prompt.md` 的内容作为系统提示词配置到AI助手中：

```bash
# 复制提示词内容到AI助手的系统提示词配置
cat ai_assistant_prompt.md
```

### 2. 在开发环境中使用

在开发MCP服务器或相关工具时，参考 `system_prompt.md` 中的详细规范：

```bash
# 查看完整规范
cat system_prompt.md
```

### 3. 自定义配置

根据具体需求修改提示词内容：

```bash
# 编辑AI助手提示词
nano ai_assistant_prompt.md

# 编辑完整系统提示词
nano system_prompt.md
```

## 核心工作流程

### 六步工作法

1. **用户意图分析** - 理解用户需求
2. **聚合搜索执行** - 收集相关信息
3. **内容梳理与总结** - 整理和结构化内容（包含内容过滤）
4. **文档格式选择** - 选择合适的输出格式
5. **Word文档生成** - 调用MCP工具生成文档
6. **服务器上传与链接生成** - 提供下载链接

### 内容安全指南

**重要提醒**：系统已集成内容过滤功能，自动避免以下敏感内容：
- 政治敏感内容（政府机构、官员、政策等）
- 宗教相关内容
- 暴力、恐怖主义相关内容
- 其他可能违反法律法规的内容

**推荐的安全内容类型**：
- 科学技术发展
- 文化艺术活动
- 教育学术研究
- 商业经济分析
- 健康医疗信息
- 环境保护话题

### MCP工具调用规范

#### 基本原则
1. **严格遵循工具定义**：使用工具文档中定义的参数名称和类型
2. **数据类型正确**：字符串参数使用双引号，数字参数不使用引号
3. **JSON格式完整**：确保JSON结构完整，括号匹配
4. **转义字符处理**：正确处理换行符、引号等特殊字符
5. **避免额外字段**：不要在参数中添加工具未定义的字段

#### 常见错误避免
- ❌ 不要在JSON中添加额外的说明文字
- ❌ 不要使用错误的参数名称
- ❌ 不要包含工具未定义的参数
- ❌ 不要使用错误的JSON格式

### 工具调用示例

#### 搜索调用
```
@Search MCP
search
query: 人工智能 医疗 应用 研究
```

#### 文档生成调用
```
@File Converter
markdown2docx
markdown_text: |
  # 人工智能在医疗领域的应用研究
  
  ## 摘要
  本文探讨了人工智能在医疗领域的应用现状...
```

**重要提醒**：调用MCP工具时，必须严格按照以下格式：
1. 使用正确的工具名称（如 `markdown2docx`）
2. 参数名称必须完全匹配（如 `markdown_text`）
3. 参数值必须是字符串类型
4. 避免在JSON中包含额外的字段或格式错误
5. 确保JSON格式完全正确，特别是转义字符

## 文档模板

### 学术报告模板
使用Markdown格式，包含摘要、引言、文献综述、研究方法、结果分析、讨论、结论、参考文献等章节。

### 商业报告模板
使用HTML格式，包含执行摘要、市场概况、竞争分析、机会与挑战、建议与展望等章节。

## 故障排除

### 常见问题

1. **搜索无结果**
   - 检查关键词是否正确
   - 尝试使用同义词
   - 扩大搜索范围

2. **文档生成失败**
   - 检查MCP工具是否正常运行
   - 验证输入格式是否正确
   - 查看错误日志

3. **下载链接无效**
   - 检查服务器是否正常运行
   - 验证文件是否成功上传
   - 确认网络连接正常

### 调试方法

1. **查看日志**
   ```bash
   # 查看MCP服务器日志
   tail -f mcp_server.log
   ```

2. **测试工具**
   ```bash
   # 测试JSON修复功能
   python test_json_fix.py
   ```

3. **验证配置**
   ```bash
   # 检查MCP配置
   cat mcp.config.json
   ```
