# API错误 (23700) 修复指南

## 错误描述
```
API错误 (23700): Agent节点执行失败(模型返回的推理内容格式不正确,无效的插件参数JSON格式)
```

## 问题分析

### 错误原因
模型在调用MCP工具时使用了错误的JSON格式：

**❌ 错误格式**：
```json
{
  "markdown_text": "# 2025年近期优质小说推荐清单\n## 🌟 综合必读榜\n..."
}
```

**✅ 正确格式**：
```
markdown_text: "# 2025年近期优质小说推荐清单\n## 🌟 综合必读榜\n..."
```

### 核心问题
1. **参数传递方式错误**：模型返回了嵌套的JSON对象，而不是直接传递字符串
2. **MCP工具解析失败**：工具期望接收直接的字符串参数，而不是JSON对象
3. **格式规范违反**：违反了MCP工具调用的标准格式

## 解决方案

### 1. 正确的MCP工具调用格式

#### Markdown转Word文档
```python
@File Converter
markdown2docx
markdown_text: |
  # 文档标题
  
  ## 章节标题
  章节内容
  
  ### 子章节标题
  子章节内容
```

#### HTML转Word文档
```python
@File Converter
html2docx
html_content: |
  <h1>文档标题</h1>
  
  <h2>章节标题</h2>
  <p>章节内容</p>
  
  <h3>子章节标题</h3>
  <p>子章节内容</p>
```

### 2. 关键要求

#### 参数名称精确匹配
- 必须使用 `markdown_text` 作为参数名（不是 `markdown` 或其他）
- 必须使用 `html_content` 作为参数名（不是 `html` 或其他）

#### 数据类型正确
- 参数值必须是字符串类型
- 直接传递markdown/HTML内容，不要包装在JSON对象中

#### 格式规范
- 使用 `|` 符号表示多行字符串
- 确保内容格式正确，转义字符处理得当
- 避免在参数中包含额外的JSON结构

### 3. 常见错误避免

#### ❌ 错误示例
```python
# 错误1：嵌套JSON对象
markdown_text: {"markdown_text": "# 标题\n内容"}

# 错误2：错误的参数名
markdown: "# 标题\n内容"

# 错误3：包含额外字段
markdown_text: "# 标题\n内容", extra_field: "value"

# 错误4：格式错误
markdown_text: "# 标题\n内容"
```

#### ✅ 正确示例
```python
# 正确格式
markdown_text: |
  # 标题
  
  ## 章节
  内容
```

### 4. 调试建议

#### 检查JSON格式
1. 确保返回的是直接的字符串参数，不是JSON对象
2. 验证参数名称完全匹配工具定义
3. 检查多行字符串格式是否正确

#### 验证工具定义
```python

@mcp.tool("markdown2docx")

    # 工具实现
```

先用简单的markdown内容测试：

### 5. 系统提示词更新

确保系统提示词包含以下关键信息：

```markdown
**MCP工具调用关键要求**：
1. **参数名称精确匹配**：必须使用 `markdown_text` 作为参数名

3. **JSON格式规范**：确保JSON结构完整，转义字符正确
4. **避免额外内容**：不要在JSON中添加说明文字或额外字段
5. **参数传递格式**：直接传递markdown字符串，不要包装在额外的JSON对象中

**⚠️ 重要提醒 - 参数传递格式**：
- ❌ 错误格式：`markdown_text: {"markdown_text": "# 标题\n内容"}`
- ✅ 正确格式：`markdown_text: "# 标题\n内容"`
```
